# Claude Code Instructions

## 중요한 지침

### ⚠️ 기존 코드 보호
- **현재 정상 작동 중인 코드는 절대 수정하지 마세요**
- 새로운 기능을 추가할 때는 기존 코드를 건드리지 않고 별도로 구현하세요
- 리팩토링이 필요한 경우 명시적으로 요청할 때만 수행하세요

### 🛡️ 안정성 우선
- 이미 테스트되고 잘 작동하는 부분은 그대로 유지
- 변경이 필요한 경우 반드시 백업 또는 주석 처리 후 진행
- 기존 기능을 해치지 않는 선에서만 개선 작업 수행

### 📝 작업 원칙
1. **추가 전용 모드**: 기본적으로 새로운 코드만 추가
2. **명시적 수정**: "수정해줘", "변경해줘"라고 명확히 요청할 때만 기존 코드 수정
3. **독립적 구현**: 새 기능은 기존 코드와 독립적으로 구현
4. **테스트 우선**: 변경 전 현재 기능이 정상 작동하는지 확인

### 🚀 현재 프로젝트 상태
- Phase 1: ✅ 완료 (기본 구조 및 UI)
- Phase 2: 🚧 진행 중 (API 통합)
- **안정적으로 작동 중인 부분**: 메인 페이지, 기본 라우팅, UI 컴포넌트

### 💡 개발 시 주의사항
```
# 기존 파일을 수정하지 말고 새 파일을 만들어서 작업
# 예: 새로운 API 클라이언트 추가 시
- ❌ 기존 파일 수정: src/lib/api.ts
- ✅ 새 파일 생성: src/lib/apis/new-api.ts
```

### 🔧 테스트 및 검증
```bash
# 변경 작업 전 항상 실행
npm run dev  # 현재 상태 확인
npm run build  # 빌드 에러 확인
npm run lint  # 린트 에러 확인
```

### 📌 보호해야 할 핵심 파일들
- `src/app/page.tsx` - 메인 페이지 (수정 금지)
- `src/app/layout.tsx` - 기본 레이아웃 (수정 금지)
- `src/components/ui/*` - shadcn/ui 컴포넌트 (수정 금지)
- 기타 정상 작동 중인 모든 파일

---

**Remember: 잘 작동하는 것은 건드리지 않는다!**
